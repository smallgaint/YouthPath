# Design System: Navy Editorial Report — Cool Lavender + Deep Navy Corporate Deck

> 추출 출처: 네이비 블루 레퍼런스 덱 8장 (01~08)
> 추출 원칙: 관찰(Observed) vs 추론(Inferred)을 분리. 콘텐츠가 아닌 디자인 문법만 기술.

---

## 1. Design Intent

- **Observed from reference:**
  - 8장 모든 페이지가 **차가운 라벤더-블루 캔버스**와 **딥 네이비 단일 강조색**을 공유한다.
  - 페이지 최상단에 **얇은 딥 네이비 풀-와이드 바**가 일관되게 존재한다 (모든 본문 페이지).
  - 본문 페이지의 타이틀 영역은 항상 **좌측 정렬 한글 대형 타이틀 + 우측 정렬 영문 소제목 + 우측 정렬 부연 설명** 3단 구조로 동일하게 반복된다.
  - 페이지 번호는 우측 하단, 작은 네이비 액센트 사각형과 짝을 이룬다.
  - 기하학적 대각선 오버레이(삼각형, 비스듬한 면)가 표지/목차/감사 페이지 등 표지성(肉) 페이지의 배경을 장식하지만, 정보 본문 페이지에서는 거의 등장하지 않는다.
  - 데이터 표현은 **다이어그램(중앙 원 + 주변 카드)**, **간트 차트형 표**, **번호 매김 카드**, **딥 네이비 정보 블록** 4개 패밀리로 한정된다.
  - 영문 라벨(PRESENTATION, CONTENTS, PROJECT OVERVIEW, PROJECT GOALS, PROPOSAL, EXPECTATION EFFECTIVENESS, MAJOR SCHEDULE, SUMMARY)이 모든 본문 페이지의 우측 상단을 헤더 라벨처럼 차지한다.

- **Inferred but not directly visible:**
  - 마스터 슬라이드는 1) 표지 / 2) 목차 / 3) 본문 / 4) (관찰 안 됨) 종결 페이지 4가지 패밀리로 구성되었을 가능성이 높다.
  - 표지의 대각선 오버레이는 하나의 동일한 모티프를 재배치하여 페이지마다 약간씩 변형한 것으로 보인다.
  - 차트 색이 거의 등장하지 않는 이유는 컬러 팔레트가 **단일 강조 + 그레이 스케일**로 의도적으로 절제되어 있기 때문으로 추론된다.
  - 폰트는 슬라이드 내 명시 텍스트("프리텐다드 ExtraLight 16")에 근거해 **Pretendard 패밀리**를 사용한 것이 거의 확실하나, 영문 라벨 폰트는 동일 패밀리의 라틴 스타일이거나 유사한 지오메트릭/네오-그로테스크 산스로 추정된다.

- **Overall impression:**
  보수적·연구보고서·사내 결재 문서 느낌. 차분한 라벤더-블루 캔버스 위에 딥 네이비가 권위감을 만들고, 화려한 컬러 대신 **타이포 위계와 카드 레이아웃**으로 정보를 정리한다. 산학 협력, 정부 과제, 컨설팅 보고, 학술 발표용으로 자연스럽게 어울리는 톤.

- **Appropriate use cases:**
  - 산학 협력·연구 과제 발표 자료
  - 정부/공공기관 제안서 및 보고서
  - 컨설팅 사내 보고용 결재 자료
  - B2B 솔루션 제안서, 임원 보고용 분기 보고서
  - 화려한 비주얼보다 신뢰·근거·구조를 우선해야 하는 분석 보고

## 2. Color System

> 모든 hex는 추출 추정치(approximate)이며, 실제 마스터 컬러와 1~3% 차이가 있을 수 있다.

- **Canvas / background:**
  - 본문 페이지 캔버스: **페일 라벤더-블루** ≈ `#E9EDF6` (approx). 순백이 아니라 약간 푸른 톤이 들어간 차가운 베이스.
  - 일부 페이지는 좌측 1/3에 별도의 라벤더 패널(`#DDE3F0` approx)을 배치한다.

- **Primary text:**
  - 딥 네이비 ≈ `#1C2C5A` (approx). 모든 타이틀, 본문 강조 텍스트, 페이지 번호, 영문 라벨에 사용.

- **Secondary text:**
  - 미디엄 블루-그레이 ≈ `#7C8AAE` (approx). 부연 설명, 보조 라벨, 카드 본문 텍스트, 비활성 표 텍스트 등 모든 약한 위계 텍스트에 사용.

- **Accent 1:**
  - 페일 라벤더 ≈ `#C9D1E5` (approx). 좌측 패널, 차트의 대각선 오버레이, 비활성 간트 바, 약한 강조 박스에 사용.

- **Accent 2 (only if clearly present):**
  - 미드 라벤더 ≈ `#A9B4D2` (approx). 간트 바의 중간 강도, 일부 보조 강조에서 관찰됨.

- **Dividers / borders:**
  - 얇은 라벤더-그레이 ≈ `#D6DCE9` (approx). 카드 외곽선, 표 그리드, 본문 구분선.

- **Chart colors:**
  - **단조로운 단일 색상 군집**: 딥 네이비 → 미드 라벤더 → 페일 라벤더 → 라벤더-그레이의 4단계 명도 변주만 사용. 다채색 팔레트는 사용하지 않음.

- **Banned / avoid:**
  - 따뜻한 색(오렌지, 빨강, 노랑) — 톤을 즉시 깬다.
  - 형광색·네온, 고채도 그라디언트.
  - 검정 본문 (#000) — 캔버스의 차가운 톤과 충돌하므로 항상 딥 네이비로 대체.
  - 4가지 이상의 차트 색상 동시 사용.

## 3. Typography System

- **Title style:**
  - 한글 대형 타이틀(예: "프로젝트 개요", "기대 효과")은 **매우 굵은 산세리프(Bold/ExtraBold)**, 딥 네이비, 좌측 정렬, 1~2줄.
  - 글자 크기는 슬라이드 폭의 약 30~45%를 차지할 정도로 크다(시각적으로 "헤드라인급").
  - 자간은 살짝 좁은 편(타이트), 줄간은 좁게 붙어있다.

- **Section header style:**
  - 영문 라벨(PROJECT OVERVIEW, MAJOR SCHEDULE 등)은 **올캡스 + 가는 자간(트래킹 살짝 넓음) + Bold 또는 SemiBold 산세리프**, 딥 네이비, 우측 상단 고정.
  - 카드 내부의 한글 소제목(기대효과 1, TEXT, CONTENT 등)은 Bold 산세리프, 딥 네이비, 좌측 정렬.

- **Body style:**
  - 한글 본문은 **얇은 산세리프(Regular 또는 Light)**, 미디엄 블루-그레이, 좌측 정렬 또는 우측 정렬.
  - 행간은 본문 위계에서 1.4~1.6 정도로 비교적 넉넉.
  - 줄당 글자수는 30~40자 내외에서 줄바꿈을 제어한다.

- **Caption / source / footnote style:**
  - 우측 상단 부연 설명: 작은 산세리프, 미디엄 블루-그레이, **우측 정렬, 2줄 고정** 패턴이 반복적으로 등장.
  - 페이지 번호: 작은 Bold 딥 네이비, 우측 하단.
  - 표지의 날짜(2080.04.15): 작은 Bold 딥 네이비, 우측 상단.

- **Numeric emphasis style:**
  - 번호 매김(01, 02, 03 …)은 **딥 네이비 채움 사각형 안에 흰색 굵은 숫자**가 들어가는 배지 형식.
  - 큰 숫자 강조는 사용되지 않음(이 덱의 절제 기조).

- **Observed casing rules:**
  - 영문은 거의 모두 **올캡스(UPPERCASE)** 사용. 문장식 케이스는 거의 없음.
  - 한글은 일반 표기. 이모지, 특수문자 거의 사용하지 않음.

- **Observed line-length behavior:**
  - 타이틀은 **2줄 이내**로 의도적으로 제한.
  - 우측 상단 부연 설명은 정확히 **2줄**로 반복적으로 맞춰져 있다.
  - 카드 본문은 3~4줄 이내가 기본.
  - **좁고 길게 6~7줄 흘러가는 헤드라인은 절대 사용하지 않는다.**

## 4. Layout Families

레퍼런스에서 직접 관찰된 본문 레이아웃 패밀리는 다음 6가지다.

- **Cover / opener (관찰됨, 01.png):**
  - 좌측 세로 회전 라벨("PRESENTATION") + 우측 상단 날짜 + 중앙 좌측 대형 한글 타이틀(2줄) + 영문 부제 + 우측 하단 부연 설명.
  - 배경에 대각선 라벤더 면(삼각형 오버레이)이 비대칭으로 깔린다.
  - 우측 하단에 작은 네이비 사각 액센트.

- **Section divider (직접 관찰 안 됨, 추론):**
  - 챕터 구분 페이지가 별도로 등장하지는 않으나, 사용한다면 표지와 동일한 대각선 오버레이 + 큰 챕터 번호 + 챕터 타이틀 형태로 구성하면 톤에 부합한다.

- **Insight / claim slide (관찰됨, 03.png · 05.png · 08.png):**
  - 상단 타이틀 + 큰 컨테이너(딥 네이비 채움 블록 또는 흰색 카드) 한 개 또는 두 개.
  - 03.png: 단일 풀-와이드 딥 네이비 블록, 내부에 흰색 텍스트 + 흰색 외곽선 박스.
  - 05.png: 좌·우 2단 흰색 카드, 각 카드 상단에 딥 네이비 헤더 스트립.
  - 08.png: 흰색 카드 안에 큰 헤드라인 + 가로 구분선 + 3단 번호 카드.

- **Chart / data slide (관찰됨, 07.png):**
  - 간트 차트 형식의 표가 페이지 본문 전체를 차지.
  - 헤더 행: 딥 네이비 채움, 흰색 텍스트(구분, 1주차~6주차).
  - 본문 행: 얇은 그리드, 좌측 항목 라벨, 우측은 다양한 명도의 라벤더/네이비 바로 일정 표시.

- **Comparison slide (관찰됨, 05.png):**
  - 좌·우 2단 평행 카드 구조. 카드 헤더에 동일한 딥 네이비 스트립 + 동일한 인용부호("") 모티프를 반복.

- **Process / timeline slide (관찰됨, 04.png · 06.png):**
  - 중앙 대형 원(딥 네이비 채움 + 페일 라벤더 외곽 링) + 주변에 카드 배치 + 카드와 원을 잇는 얇은 연결선/화살표.
  - 04.png: 4-사분면 (2x2) 카드 배치.
  - 06.png: 좌측 부연 설명 영역 + 중앙 원 + 우측 3단 세로 카드 배치.

- **Closing / CTA slide (직접 관찰 안 됨, 추론):**
  - 본 레퍼런스 8장에는 종결 페이지가 포함되지 않음. 사용 시 표지와 동일한 대각선 오버레이 위에 "감사합니다" 류의 짧은 메시지 + 하단 풀-와이드 구분선 + 발표자 정보·연락처를 좌·중·우 3등분 배치하는 형식이 톤에 가장 잘 부합한다.

## 5. Flow Architecture

- **Title page flow:**
  - 최상단 풀-와이드 네이비 바 → 좌측 세로 회전 라벨 → 우측 상단 날짜 → 중앙 영역 대형 타이틀(한글 2줄 + 영문 부제) → 배경에 비대칭 대각선 오버레이 → 우측 하단 부연 설명 + 작은 네이비 사각 액센트.

- **Body page flow:**
  - 최상단 네이비 바 → 좌측 상단 한글 대형 타이틀 → 우측 상단 영문 라벨(올캡스) + 우측 정렬 부연 설명(2줄) → 중앙 본문 영역(레이아웃 패밀리에 따라 변화) → 우측 하단 페이지 번호 + 작은 네이비 사각 액센트.

- **End page flow:**
  - 본 레퍼런스 미관찰. 사용 시 표지 모티프(대각선 오버레이) 재활용 권장.

- **Header / body / footer structure:**
  - **Header zone (상단 약 12~15%):** 풀-와이드 네이비 바 + 한글 대형 타이틀 + 영문 라벨 + 부연 설명.
  - **Body zone (중간 약 70~75%):** 콘텐츠 패밀리에 따라 변화. 딥 네이비 블록·흰색 카드·중앙 원·간트 표 4가지 중 하나.
  - **Footer zone (하단 약 5~8%):** 우측 하단에 페이지 번호와 작은 네이비 사각 액센트. 그 외 영역은 비워둔다.

- **Header zone placement rules:**
  - 한글 타이틀: 좌측 정렬, 좌측 마진에서 시작.
  - 영문 라벨: 우측 정렬, 우측 마진에서 끝남.
  - 부연 설명: 영문 라벨 바로 아래, 우측 정렬, 정확히 2줄.

- **Body zone placement rules:**
  - 카드/블록의 좌·우 마진은 헤더 영역의 좌·우 마진과 일치하여 수직 정렬을 유지.
  - 중앙 원 모티프(다이어그램 슬라이드)는 페이지 정중앙 또는 약간 좌측에 배치.

- **Footer zone placement rules:**
  - 페이지 번호는 항상 우측 하단 동일 위치.
  - 페이지 번호 좌측에 작은 네이비 사각 액센트(약 4~6px)를 거의 항상 동반.
  - 좌측 하단은 의도적으로 비워두어 시선의 균형을 잡는다(예외: 표지에서 부연 설명이 들어옴).

## 6. Grid, Alignment, and Spacing

- **Outer margins:**
  - 좌·우 마진은 슬라이드 폭의 약 5~7% 수준으로 비교적 좁고, 콘텐츠 영역이 넓게 활용된다.
  - 상단 마진은 네이비 바 바로 아래에서 시작.
  - 하단 마진은 페이지 번호까지 약간의 여백을 두고 끝남.

- **Column behavior:**
  - 1단(풀-와이드) / 2단(좌-우 균등) / 비대칭 2단(좌측 1/3 + 우측 2/3) 세 가지가 명확히 반복.
  - 02.png(목차)는 좌측 1/3 라벤더 패널 + 우측 2/3 콘텐츠 구조.
  - 06.png는 좌측 1/3 + 중앙 1/3(원) + 우측 1/3(카드 3단) 구조.

- **Text alignment:**
  - 한글 타이틀: 좌측 정렬.
  - 영문 라벨: 우측 정렬.
  - 부연 설명: 우측 정렬(우측 상단), 좌측 정렬(좌측 영역에 배치된 경우).
  - 카드 본문: 좌측 정렬이 기본.
  - 표 헤더: 가운데 정렬.
  - 표 본문: 좌측 정렬(라벨), 가운데 정렬(바 안 텍스트).

- **Whitespace philosophy:**
  - **여백을 적극적으로 활용**한다. 빈 영역을 두려워하지 않으며, 좌측 하단처럼 의도적으로 비워두는 영역이 거의 모든 본문 페이지에 존재.
  - 카드와 카드 사이의 간격은 카드 자체의 약 8~12% 수준으로 비교적 넉넉.

- **Density level:**
  - **중간 밀도**. 보고서스럽게 정보를 채우되, 답답하지 않게 카드와 여백으로 호흡을 만든다.

- **Object anchoring rules:**
  - 모든 본문 페이지 요소는 **좌·우 마진과 헤더 라인 / 푸터 라인**의 격자에 정렬된다.
  - 중앙 원 같은 다이어그램 요소는 페이지 정중앙 수직축을 기준으로 대칭 배치.
  - 우측 상단 영문 라벨과 우측 하단 페이지 번호는 동일한 우측 마진에 수직으로 정렬되어, 페이지 우측에 보이지 않는 정렬축이 형성된다.

## 7. Components

- **Title block:**
  - 매우 굵은 한글 산세리프, 딥 네이비, 1~2줄, 좌측 상단.
  - 옆 또는 아래에 영문 부제(작은 그레이-블루)가 동반될 수 있음.

- **Subtitle / kicker:**
  - 우측 상단 영문 올캡스 라벨이 모든 본문 페이지에서 키커 역할.
  - 표지에서는 좌측 세로 회전 라벨("PRESENTATION")이 추가 키커로 작동.

- **Bullets / key points:**
  - 작은 원형 불릿(딥 네이비 채움 또는 라벤더 채움) + 한글 텍스트.
  - 중첩 불릿은 거의 사용하지 않음(레벨 1만).

- **Cards / callouts:**
  - **흰색 카드**: 얇은 라벤더-그레이 외곽선, 내부에 헤더 + 본문 + (선택) 구분선 + 불릿 리스트.
  - **딥 네이비 채움 블록**: 흰색 텍스트, 흰색 가는 외곽선의 보조 박스, 안에서 다시 카드/박스 구조 가능.
  - **인용 카드**: 큰 따옴표 모티프(") 좌측 상단에 두고 본문을 아래에 흘려보냄.

- **Tables:**
  - 헤더 행: 딥 네이비 채움 + 흰색 굵은 텍스트 + 가운데 정렬.
  - 본문 행: 흰색 또는 매우 옅은 라벤더 배경 + 얇은 라벤더-그레이 그리드.
  - 표 외곽선은 거의 없거나 매우 얇음.

- **Charts:**
  - 본 레퍼런스에서 막대/선/산점도 차트는 직접 등장하지 않으나, 사용한다면:
    - 단색 군집(딥 네이비 → 페일 라벤더 명도 변주만 사용).
    - 그리드는 매우 얇은 라벤더-그레이.
    - 축 라벨은 미디엄 블루-그레이 산세리프.
    - 데이터 라벨은 막대 위 또는 옆에 작은 딥 네이비 텍스트.

- **Legends / labels:**
  - 가로 1줄, 작은 산세리프, 차트 상단 또는 하단에 짧게.

- **Icons / illustrations / photography:**
  - 06.png에서 **얇은 선형(라인) 아이콘**(타깃, 전구, 악수)이 카드 우측 상단에 작게 등장.
  - 아이콘은 모두 동일한 라인 굵기·동일한 라벤더 톤으로 통일.
  - 05.png에서 사진(노트북, 화분)이 매우 옅은 회색조로 페이지 하단을 가로지르며 깔리지만, 거의 워터마크 수준의 낮은 명도/대비.

- **Icon placement and usage rules:**
  - 아이콘은 항상 **카드의 우측 상단 모서리** 근처에 작게 배치.
  - 아이콘은 콘텐츠를 설명하는 **보조 단서** 역할이며, 아이콘이 정보를 단독으로 전달하지는 않음.
  - 아이콘 색은 딥 네이비 또는 미드 라벤더 라인 단색.

- **Infographic cards / metric cards:**
  - 번호 배지(01, 02, 03 …) + 한글 소제목 + 한글 본문 패턴이 반복되는 카드형 인포그래픽.
  - 카드는 흰색 배경에 얇은 외곽선(또는 외곽선 없이 그림자만), 좌측 상단에 번호 배지가 살짝 카드 외곽을 침범하듯 걸쳐있는 구조.

- **Diagram / flow modules:**
  - 중앙 원(허브) + 주변 카드(스포크) 구조가 두 번 등장(04.png, 06.png).
  - 원과 카드를 잇는 **얇은 연결선(직선)** 또는 **얇은 화살표**가 일관되게 사용됨.
  - 연결선은 색상/굵기가 모두 통일.

## 8. Data Visualization Language

- **Preferred chart families:**
  - **표 + 간트 차트형 일정표**가 데이터 시각화의 주된 표현.
  - **허브-스포크 다이어그램**(중앙 원 + 카드)이 개념 시각화의 주된 표현.
  - 막대/선/원/산점도 등 정량 차트는 본 레퍼런스에서 등장하지 않으나, 사용 시 단색 군집 원칙을 따른다.

- **Axis / gridline treatment:**
  - 표는 매우 얇은 라벤더-그레이 그리드.
  - 차트 축이 등장한다면 0축(영선)만 살짝 강조하고 보조 그리드는 거의 보이지 않게.

- **Labeling style:**
  - 표 헤더는 딥 네이비 채움 + 흰색.
  - 표 본문 라벨은 미디엄 블루-그레이.
  - 막대/바 내부 텍스트는 흰색 또는 딥 네이비 산세리프.

- **Annotation style:**
  - 별도의 주석 모듈은 등장하지 않음.
  - 사용한다면 작은 미디엄 블루-그레이 텍스트 + 얇은 연결선으로 미니멀하게.

- **When to avoid charts:**
  - 정성적 메시지(개념·관계·구조)를 전달할 때는 차트 대신 **카드 + 다이어그램** 표현을 우선.
  - 데이터가 3~4 포인트밖에 없을 때는 차트 대신 **번호 카드** 또는 **숫자 강조 박스**를 사용.

- **Infographic composition style:**
  - 카드 그리드(2x2 또는 3-칼럼)가 기본.
  - 카드 내부는 항상 **번호 + 헤딩 + 바디** 3단 위계를 유지.

- **Icon-led data communication:**
  - 아이콘은 메트릭을 보조하는 시각적 단서로만 사용.
  - 아이콘 단독으로 수치를 전달하지 않음.
  - 아이콘 + 라벨 + 짧은 설명 조합이 표준.

- **Diagram flow direction and connector behavior:**
  - 허브-스포크 구조에서 화살표 방향은 **중심 → 카드(분기형)** 또는 **카드 → 중심(수렴형)** 둘 중 하나로 슬라이드 안에서 일관되게 통일.
  - 연결선은 항상 직선 또는 매우 완만한 곡선이며, 굴절이 많은 폴리라인은 사용하지 않음.

## 9. Imagery and Graphic Treatment

- **Image crop / masking:**
  - 사진은 거의 사용하지 않으며, 사용 시 **저채도·저대비**로 처리해 워터마크 수준에 가깝게.
  - 사진은 페이지의 일부 영역에만 깔리며 본문 콘텐츠를 가리지 않는다.

- **Use of gradients / fills:**
  - 그라디언트는 거의 사용하지 않음.
  - 사용한다면 페일 라벤더 → 약간 더 짙은 라벤더로 미세하게(거의 단색에 가깝게).

- **Use of shapes / panels / bands:**
  - **풀-와이드 네이비 바**(상단), **딥 네이비 채움 사각 블록**(본문), **좌측 1/3 라벤더 패널**, **헤더 스트립**(카드 상단의 짙은 배너) 등 **사각형 기반의 절제된 면 분할**이 핵심.
  - 표지/감사 페이지에 한해 **비대칭 대각 삼각형 오버레이**가 등장.

- **Use of texture / shadows:**
  - 그림자는 거의 사용하지 않음(매우 얕은 카드 그림자는 가능).
  - 텍스처(노이즈, 그레인)는 사용하지 않음. 깔끔한 평면 디자인.

## 10. Slide-System Rules

- **What repeats across most slides:**
  - 최상단 풀-와이드 네이비 바.
  - 좌측 상단 한글 대형 타이틀 + 우측 상단 영문 올캡스 라벨 + 우측 정렬 부연 설명(2줄).
  - 우측 하단 페이지 번호 + 작은 네이비 사각 액센트.
  - 단색 라벤더-네이비 팔레트.
  - Pretendard 패밀리 (또는 동급 한글 산세리프).

- **What should vary cautiously:**
  - 본문 영역의 레이아웃 패밀리(딥 네이비 블록, 흰색 카드, 중앙 원 다이어그램, 간트 표 4종 중 택일).
  - 카드의 개수(2/3/4/6).
  - 좌측 라벤더 패널의 유무(목차·일부 강조 페이지에만 등장).

- **Body-slide layout discipline:**
  - 본문 페이지는 **헤더 구조를 절대 바꾸지 않는다.** (한글 타이틀 좌측 정렬 + 영문 라벨 우측 상단 + 부연 설명 우측 2줄)
  - 본문 영역에서만 4가지 레이아웃 패밀리 중 하나를 선택해 변주.
  - 같은 덱 안에서 동일한 레이아웃 패밀리는 **시각적 차이가 거의 없도록 통일**해야 한다(예: 카드 크기, 번호 배지 크기, 헤더 스트립 두께가 페이지마다 들쭉날쭉하면 안 됨).

- **What must remain consistent across the deck:**
  - 컬러 팔레트(절대 새 강조색을 추가하지 않는다).
  - 폰트 패밀리.
  - 헤더/푸터 위치와 구조.
  - 번호 배지 형식(딥 네이비 사각 + 흰색 숫자).
  - 페이지 번호 우측 하단 배치.

- **What stays consistent across title/body/end pages:**
  - 컬러 팔레트와 폰트.
  - 우측 하단 페이지 번호 + 작은 네이비 사각 액센트(표지 제외).
  - 대각선 오버레이는 **표지와 종결 페이지**에만 등장(본문 페이지에서는 등장하지 않음).

- **How icons/infographics should repeat across the deck:**
  - 아이콘은 동일한 라인 두께, 동일한 단색(딥 네이비 또는 미드 라벤더), 동일한 크기로 통일.
  - 인포그래픽 카드는 전 페이지에서 동일한 외곽선 두께, 동일한 모서리 둥글기, 동일한 헤더 스트립 두께를 유지.
  - 다이어그램의 연결선·화살표 굵기와 색상은 페이지마다 동일.

## 11. Anti-Patterns

다음은 본 디자인 시스템에서 **반드시 피해야 할 패턴**이다.

- 따뜻한 강조색(빨강, 주황, 노랑) 또는 형광 컬러를 추가하는 것 — 차가운 톤의 권위감을 즉시 깬다.
- 다채색 차트 팔레트(빨·파·노·초 동시 사용) — 단색 명도 변주 원칙 위반.
- 좁고 길게 6~7줄로 흘러가는 헤드라인 — 타이틀은 1~2줄로 제한해야 한다.
- 본문 페이지에 표지의 대각선 오버레이를 끌어오는 것 — 본문은 깔끔한 캔버스를 유지해야 한다.
- 헤더 영역의 영문 올캡스 라벨을 빼거나 위치를 바꾸는 것 — 모든 본문 페이지의 시각적 리듬이 깨진다.
- 페이지마다 카드 모양·외곽선 두께·헤더 스트립 두께를 다르게 만드는 것 — 일관성이 핵심.
- 의미 없이 반짝이는 배지/뱃지 스팸 — 번호 배지는 카운트가 명확한 경우에만 사용.
- 의미 없이 둥둥 떠 있는 아이콘 — 아이콘은 항상 카드 우측 상단에 정렬되며, 의미적 역할(개념 단서)을 가져야 한다.
- 사진을 풀-블리드로 깔거나 고채도로 사용하는 것 — 사진은 워터마크 수준의 저채도로만.
- 화려한 그라디언트, 그림자, 텍스처 — 평면 디자인 원칙을 깨뜨린다.
- 페이지 번호를 좌측 또는 가운데로 이동하는 것 — 우측 하단 고정.
- 같은 덱 안에서 한 슬라이드는 허브-스포크, 다른 슬라이드는 자유 곡선 다이어그램으로 다이어그램 문법이 섞이는 것 — 한 가지 다이어그램 문법으로 통일.
- 영문과 한글에 서로 다른 폰트 패밀리를 섞어 쓰는 것 — Pretendard 또는 동일 패밀리로 통일.
- 본문 텍스트에 순흑(#000)을 사용하는 것 — 항상 딥 네이비로 대체.
